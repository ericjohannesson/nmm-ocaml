OPTIMAL PARTITIONING

Eric Johannesson

§ 1

Introduction

Suppose you have a file system such that, whenever you modify a file, the modification time of its immediate parent directory also gets updated. Furthermore, suppose you want to keep a directory in your file system synced with another directory in your file system.

Due to the aforementioned property of your file system, you do not need to compare the modification time of every file in your source directory with that of its counterpart in the destination directory. It is sufficient to compare the modification time of files that reside in subdirectories whose modification times differ.

Now, in order to optimize the syncing procedure, how many subdirectories should you have? It is obvious that having no subdirectories is equivalent to having one subdirectory for each file, in the sense that it will require equally many comparisons in each case (as many comparisons as there are files, to be precise). Presumably, there is an optimum somewhere in between.

§ 2

A probabilistic approach

Assume, for the sake of simplicity, that

─

you have n files partitioned equally over k subdirectories (with n/k files in each),

─

each file is just as likely to have been modified as any other (with probability p),

─

modifications are probabilistically independent of each other,

─

the number of files are fixed, and the modification time of a directory only gets updated when its files are modified.

For each subdirectory, the probability of its modification time not having been updated is thus given by

(a)
q = (1 - p)^(n/k)

The expected number of comparisons that need to be performed is then given by the following sum:

(b)
k + q^(k - 0) × (1 - q)^0 × k!/0!/(k - 0)! × 0 × n/k
+
q^(k - 1) × (1 - q)^1 × k!/1!/(k - 1)! × 1 × n/k
+
.
.
.
+
q^(k - i) × (1 - q)^i × k!/i!/(k - i)! × i × n/k
+
.
.
.
+
q^(k - k) × (1 - q)^k × k!/k!/(k - k)! × n × n/k

For a given p and n, we want to find a k that minimizes (b). Provided that n is sufficiently small, we can always do so using numerical brute force methods. There may also be a way of approximating (b) that makes it amenable to analytical methods, but let us consider an alternative approach instead.

§ 3

A non-probabilistic approach

Assume, as before, that

─

you have n files partitioned equally over k subdirectories (with n/k files in each), and that

─

the number of files are fixed, and the modification time of a directory only gets updated when its files are modified.

Assume, moreover, that

─

exactly one file has been modified.

In the worst case scenario, the number of comparisons that need to be performed is then given by

(a)
k + n/k

The derivative of (a) with respect to k is given by

(b)
1 - n/k²

For a given n, to minimize (a) with respect to k is simply a matter of finding the root of (b) with respect to k, i.e. to solve the following equation for k:

1 - n/k² = 0

The solution is given by

k = √n
